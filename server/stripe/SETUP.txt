STRIPE VERIFY BACKEND — FINAL SETUP (VERCEL + DHOSTING FRONTEND)
=================================================================

DOCZELOWA ARCHITEKTURA
----------------------
- Frontend (statyczny): dhosting
- Backend weryfikacji Stripe: Vercel (Node serverless)
- Docelowy URL backendu: https://payments.relationshipscan.app

To jest URL, który ma być ustawiony w:
- success.html
- result.html
- report.html
meta: stripe-verify-api-base

1) CO JUZ JEST PRZYGOTOWANE W REPO
----------------------------------
- Backend: server/stripe/index.mjs
- Konfiguracja env: server/stripe/.env.example
- Render blueprint: render.yaml (root repo)
- Frontend meta ustawione na:
  https://payments.relationshipscan.app

2) DEPLOY NA VERCEL
-------------------
1. W Vercel ustaw Root Directory = server/stripe
2. W Vercel env ustaw:
   - STRIPE_SECRET_KEY
   - STRIPE_WEBHOOK_SECRET
   - FRONTEND_URL=https://relationshipscan.app
   - ACCESS_GRANT_TTL_HOURS=24
   - TIKTOK_PIXEL_ID=D7NR5V3C77UCPV3KTR80
   - TIKTOK_ACCESS_TOKEN=<token z TikTok Events API>
3. Ustaw custom domain:
   https://payments.relationshipscan.app

3) DNS W DHOSTING
-----------------
W panelu DNS domeny relationshipscan.app dodaj rekord:
- Type: CNAME
- Host: payments
- Value: target podany przez Vercel dla custom domain

Po propagacji:
https://payments.relationshipscan.app/health
ma zwracać JSON z ok:true.

4) WEBHOOK STRIPE (PRODUKCJA) — JEDEN ENDPOINT
-----------------------------
W Stripe Dashboard:
Developers > Webhooks > Add endpoint

Endpoint URL:
https://payments.relationshipscan.app/api/stripe/webhook

Zdarzenia:
- checkout.session.completed
- checkout.session.async_payment_succeeded
- payment_intent.succeeded
- payment_intent.payment_failed

Webhook destinations:
- ZOSTAW: thin payload
- USUŃ: snapshot payload

Backend używa tylko jednego sekretu:
STRIPE_WEBHOOK_SECRET

5) CO WERYFIKUJE BACKEND
------------------------
POST /api/stripe/confirm-return
body: { "session_id": "...", "payment_intent": "..." }

Weryfikacja po Stripe API:
- checkout.session.payment_status == paid
  oraz/lub
- payment_intent.status == succeeded

Dopiero wtedy backend zwraca grantToken.

6) TIKTOK EVENTS API (SERVER-SIDE)
----------------------------------
Backend wysyła event "CompletePayment" do TikTok Events API po potwierdzeniu płatności:
- webhook: checkout.session.completed / checkout.session.async_payment_succeeded (status paid)
- confirm-return: fallback po udanym verify

Wymagane ENV:
- TIKTOK_PIXEL_ID
- TIKTOK_ACCESS_TOKEN

Token trzymaj wyłącznie po stronie backendu (Vercel ENV), nigdy w JS frontendu.

7) KONTROLA DOSTEPU W FRONTEND
------------------------------
- Front zapisuje wyłącznie stateless grantToken podpisany po stronie backendu.
- Każde wejście na report/result sprawdza:
  GET /api/stripe/grant-status?grant=...
- Bez aktywnego grantu nie ma odblokowania premium.
